### Design Specifications

**Header Overlap Fix:**
- Current: `pt-20` (5rem / 80px)
- Navigation header actual height: 
  - Padding: `py-5 sm:py-6` = 20px/24px top + 20px/24px bottom = 40px/48px
  - Content height: ~30-40px (logo + nav links)
  - Total: ~70-88px depending on screen size
- Issue: Large title (`text-4xl sm:text-5xl md:text-6xl lg:text-7xl`) visually appears to overlap even with 80px padding
- Recommended solutions:
  1. **Increase padding:** `pt-24 sm:pt-28` (96px/112px) - provides safe margin
  2. **Add margin to hero:** Add `mt-4 sm:mt-6` to hero section for extra breathing room
  3. **Reduce title size slightly:** Consider `text-3xl sm:text-4xl md:text-5xl lg:text-6xl` if overlap persists
  4. **Best approach:** Combine `pt-24` with `mt-4` on hero section for optimal spacing

**Button Layout Fix:**
- Current: `flex flex-wrap gap-3` with `size="sm"` buttons (`px-4 py-2 text-sm`)
- Issues identified:
  1. Cards are in 3-column grid on desktop, making each card narrow (~33% width)
  2. Multiple buttons (GitHub, Demo, Article, Watch Video) compete for space
  3. Buttons may overflow card boundaries or overlap
  4. Gap of `gap-3` (12px) may be insufficient
  5. Buttons need better flex properties to prevent overflow
- Recommended solutions:
  1. **Increase gap:** `gap-3 sm:gap-4` (12px mobile, 16px desktop)
  2. **Better flex properties:** Add `min-w-0 flex-shrink` to buttons to allow proper wrapping
  3. **Responsive button sizing:** Consider even smaller buttons on mobile or in cards
  4. **Button container:** Ensure container has `w-full` and proper overflow handling
  5. **Alternative layout:** Consider vertical stacking on very narrow cards or mobile
  6. **Button text:** Keep button labels short (already done: "GitHub", "Demo", "Article")

**Recommended Solutions:**

1. **Header Overlap (Best Approach):**
   ```tsx
   // In app/projects/page.tsx
   <main id="main-content" className="min-h-screen pt-24 sm:pt-28">
     <section className="flex items-center justify-center px-6 py-16 sm:py-20 mt-4 sm:mt-6">
       {/* Hero content */}
     </section>
   </main>
   ```
   - `pt-24 sm:pt-28` provides 96px/112px top padding (safe margin above header)
   - `mt-4 sm:mt-6` adds extra breathing room between padding and content
   - This ensures large title never visually overlaps

2. **Button Layout (Best Approach):**
   ```tsx
   // In components/ProjectCard.tsx
   <div className="flex flex-wrap gap-3 sm:gap-4 w-full">
     {github && (
       <Button
         variant="primary"
         size="sm"
         href={github}
         external
         className="flex-shrink min-w-0"
       >
         GitHub
       </Button>
     )}
     {/* ... other buttons */}
   </div>
   ```
   - `gap-3 sm:gap-4` provides better spacing (12px mobile, 16px desktop)
   - `w-full` ensures container uses full width
   - `flex-shrink min-w-0` on buttons allows proper wrapping
   - Consider: If buttons still overlap, reduce to `gap-2 sm:gap-3` or use vertical stack on mobile
