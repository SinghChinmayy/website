## 5. Technical Requirements

### Functional Requirements
- Button component supports `variant` prop: `"primary"` | `"secondary"` | `"ghost"`
- Button component supports `size` prop: `"sm"` | `"md"` | `"lg"` (default: `"md"`)
- Button can render as:
  - `<button>` when no href provided
  - Next.js `Link` component for internal routes (href starts with `/`)
  - `<a>` tag for external links (when `external={true}` or href is external URL)
- Button supports `disabled` state with visual feedback and prevents interaction
- Button supports `loading` state (optional, for future use)
- Button accepts all standard button/anchor attributes (onClick, aria-label, etc.)
- Button works seamlessly with Framer Motion wrapper components (when wrapped in motion.div)
- Button maintains all existing aria-labels from current implementation

### Non-Functional Requirements
- **Performance:** No additional bundle size beyond necessary styles
- **Accessibility:** WCAG 2.1 AA compliant focus states, proper ARIA attributes
- **Usability:** Buttons must have clear hover/active/focus states
- **Responsive Design:** Buttons scale appropriately on mobile
- **Theme Support:** Full dark mode support

### Technical Constraints
- Must use Tailwind CSS for styling (no additional CSS-in-JS libraries)
- Must maintain compatibility with existing Framer Motion animations
- Should follow project's existing component patterns
- Use Next.js `Link` component for internal navigation (not regular `<a>` tags)
- No external UI libraries (keep it lightweight)
- Consider using `clsx` or similar utility for className merging if needed (check if already in project)

## 8. Frontend Changes

### New Components

**Create: `components/Button.tsx`**

A polymorphic button component with the following features:

```tsx
// Proposed API
interface ButtonProps {
  variant?: 'primary' | 'secondary' | 'ghost';
  size?: 'sm' | 'md' | 'lg';
  href?: string;  // If provided, renders as Next.js Link (internal) or <a> (external)
  external?: boolean;  // If href is external URL, set to true
  disabled?: boolean;
  loading?: boolean;
  children: React.ReactNode;
  className?: string;
  // ...rest of button/anchor props (onClick, aria-label, etc.)
}

// Usage examples:
<Button variant="primary">View Projects</Button>
<Button variant="secondary" href="/contact">Contact Me</Button>
<Button variant="ghost" href="https://github.com" external>GitHub</Button>
<Button variant="primary" disabled>Processing...</Button>
<Button variant="primary" loading>Submitting</Button>
```

**Important Implementation Notes:**
- Use Next.js `Link` component for internal routes (href starts with `/`)
- Use regular `<a>` tag for external links (with `rel="noopener noreferrer"`)
- Use `<button>` element when no href is provided
- Support disabled and loading states with appropriate visual feedback

**Design Specifications (Industry Standard):**

| Variant   | Background | Text | Border | Shadow | Hover Effect |
|-----------|------------|------|--------|--------|--------------|
| Primary   | Solid fill | Contrasting | None | Subtle | Slight lift + deeper shadow |
| Secondary | Transparent/Subtle | Theme color | Subtle 1px | None | Light fill on hover |
| Ghost     | Transparent | Theme color | None | None | Underline or subtle bg |

**Size Specifications:**

| Size | Padding | Font Size | Min Width |
|------|---------|-----------|-----------|
| sm   | px-4 py-2 | text-sm | 80px |
| md   | px-6 py-2.5 | text-base | 100px |
| lg   | px-8 py-3 | text-lg | 120px |

## 14. Design Reference

### Modern Button Design Principles

1. **Subtle Shadows** - Use `shadow-sm` on primary buttons, `shadow-md` on hover
2. **Smooth Transitions** - `transition-all duration-200 ease-out`
3. **Rounded Corners** - `rounded-lg` or `rounded-xl` for modern feel
4. **No Heavy Borders** - Remove `border-2`, use `border` or no border for primary
5. **Hover Lift** - Slight translateY on hover for interactive feel
6. **Color Contrast** - Ensure 4.5:1 contrast ratio minimum

### Example Primary Button Style
```css
/* Light Mode */
bg-gray-900 text-white 
shadow-sm hover:shadow-md 
hover:-translate-y-0.5
rounded-xl
transition-all duration-200 ease-out

/* Dark Mode */
dark:bg-white dark:text-gray-900
```

### Example Secondary Button Style
```css
/* Light Mode */
bg-gray-100 text-gray-900
hover:bg-gray-200
rounded-xl
transition-all duration-200 ease-out

/* Dark Mode */
dark:bg-gray-800 dark:text-white
dark:hover:bg-gray-700
```

### Example Ghost Button Style
```css
/* Light Mode */
bg-transparent text-gray-900
hover:bg-gray-100
border border-gray-200
rounded-xl
transition-all duration-200 ease-out

/* Dark Mode */
dark:text-white
dark:hover:bg-gray-800
dark:border-gray-700
```

### Disabled State Style
```css
/* All variants when disabled */
opacity-50 cursor-not-allowed pointer-events-none
/* Remove hover effects when disabled */
```
