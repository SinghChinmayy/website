## 3. Context & Problem Definition

### Problem Statement
The site currently has misaligned sections that create an inconsistent visual hierarchy:

1. **Header:** Navigation text is centered, but the layout is full-width and unstructured. Elements float across the full viewport without proper containment.

2. **Body Content:** Content appears visually centered but is not structurally aligned with header and footer. Uses individual max-width containers instead of a shared system.

3. **Footer:** Content is misaligned relative to header and body, creating a disconnected appearance.

This results in "fake center alignment" - the site looks centered but doesn't have true structural alignment. Each section operates independently with its own alignment logic, causing visual inconsistencies.

## 15. Specific Implementation Details

### Container Class Implementation

**Option 1: Tailwind @apply (RECOMMENDED for Tailwind v4)**
```css
@layer components {
  .container {
    @apply w-full max-w-6xl mx-auto px-4 sm:px-6;
  }
}
```
**Why this is best:**
- âœ… Uses Tailwind utilities via `@apply` (better framework integration)
- âœ… Respects Tailwind's configuration and theme
- âœ… Easier to maintain and modify
- âœ… Consistent with Tailwind v4 best practices
- âœ… `max-w-6xl` = 1152px (within 1100-1200px requirement)
- âœ… Responsive padding built-in (`px-4` mobile, `px-6` desktop)

**Option 2: Inline Tailwind Utilities (Alternative if you prefer)**
Use directly in components:
```tsx
className="w-full max-w-6xl mx-auto px-4 sm:px-6"
```
**Note:** This is less DRY but acceptable if you prefer utility-first approach.

**Option 3: Custom CSS (Not Recommended)**
Only use if you need custom behavior not available in Tailwind:
```css
.container {
  width: 100%;
  max-width: 1200px;
  margin-left: auto;
  margin-right: auto;
  padding-left: 1rem;
  padding-right: 1rem;
}

@media (min-width: 640px) {
  .container {
    padding-left: 1.5rem;
    padding-right: 1.5rem;
  }
}
```
