### Naming Rules
When creating a new task file from this template, follow these naming conventions:

1. **Check Existing Files:** Before creating a new task file, check the `ai_docs/tasks/` folder to find the highest serial number currently in use.

2. **File Naming Format:** Use the format `N.<what_is_this_task_about>.md` where **N** is the next sequential number (incremented for each new task), and `<what_is_this_task_about>` is a concise, underscored description of the main goal of the task.
   - Example: `1.fix_dark_mode.md`, `2.modify_css.md`, etc.

3. **File Location:** All task files must be placed in the `ai_docs/tasks/` folder.

4. **Serial Numbering:** 
   - Do NOT use generic names like `task_1.md`, `task_2.md`, etc.
   - Always use the format above, incrementing N to the next highest task number present in the folder.
   - If no task files exist, start with `1.<what_is_this_task_about>.md`.
